feat: Phase 1 - Library/Binary Architecture Refactoring

BREAKING: None (fully backwards compatible)

This commit addresses architecture issues identified by Copilot regarding
improper mixing of I/O operations in library code. Implements Phase 1 of
a phased refactoring approach.

## What Changed

### Created Binary-Only Terminal Module
- Added src/terminal/ module (NOT exposed in lib.rs)
  - mod.rs: Terminal utilities and size detection
  - display.rs: Print functions (success, error, warning, info, separators)
  - input.rs: User input (confirm, prompt)
  - progress.rs: Progress bars and spinners

### Added Deprecation Warnings
- Marked 177 I/O functions with #[deprecated] in:
  - src/cli/mod.rs (print functions, spinners, terminal width)
  - src/display/mod.rs (clear_screen, print_separator, print_centered)
- Clear deprecation messages: "Move to binary crate - performs I/O"

### Created Pure Function Alternatives
- format_table_with_width() - pure table formatter
- format_separator() - returns String (no printing)
- format_centered() - returns String (no printing)
- format_list() - formats lists without I/O
- format_key_value_list() - formats key-value pairs without I/O

### Documentation
- Added comprehensive architecture notes to src/lib.rs
- Created docs/architecture-refactor.md (full technical details)
- Created REFACTORING_CHECKLIST.md (work tracking)
- Created REFACTORING_SUMMARY.md (executive summary)

### Bug Fixes
- Fixed missing rand::Rng import in src/display/tips.rs
- Made get_random_tips() and get_random_tip() public
- Fixed re-exports in src/display/mod.rs

## Why This Matters

Libraries with I/O operations cannot be reused in:
- GUI applications
- Web services
- Testing environments (without mocking)
- Other libraries
- Embedded systems
